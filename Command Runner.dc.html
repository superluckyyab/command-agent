使用手册 — 文件目录与远程传输

Files 目录让你把脚本和工具放在一起，用 PowerShell 直接调用；远程传输已作为 File Transfer 命令类型集成到 Commands 中。

1. 目录在哪里 · 怎么放文件
  • 目录是程序 exe 同级的 files 文件夹（首次运行自动创建）。在 Files 页顶部能看到完整路径。
  • 点 Open folder 打开该文件夹，把 .ps1 / .bat / .exe / 数据文件拖进去，再点 Refresh 刷新列表。
2. PowerShell 怎么调用目录里的文件

每次执行命令时，程序会自动：把工作目录切到该文件夹、把它加进 PATH、并设置变量 $Dir 指向它。所以在任意命令 / 脚本 / Shortcut 里都可以这样调用：

.\deploy.ps1 # 运行目录里的脚本 & "$Dir\tool.exe" --version # 用绝对路径运行工具 Get-Content .\config.txt # 读取目录里的数据文件 mytool.exe # 目录已在 PATH，exe 可直接按名调用
3. 怎么运行一个文件
  • 在 Files 列表里点该文件的 Run，输出会打印到 Console。
  • 或在命令里写 & ".\文件名"（例如 & ".\backup.ps1"）。
4. SSH 远端命令 / 文件传输（plink & pscp）
  • 程序自带 tools/plink.exe 和 tools/pscp.exe，并把 tools 目录加进 PATH。
  • 添加 SSH 类型命令时，程序用 plink -ssh user@host -P port -pw pass -batch "命令" 登录执行。
  • 添加 File Transfer 类型命令时，程序用 pscp -pw pass -P port ".\文件" user@host:"/远端目录/" 上传。
  • 首次连接新主机时，程序会先 pipe 一个 yes 来自动接受主机密钥；失败可手动在 Console 执行一次确认。
  • 如果 bundled 工具被删除，会回退到系统 PATH 中的 plink / pscp。

等价的手动命令（可直接在 PowerShell 里执行）：

& "$Dir\..\tools\plink.exe" -ssh user@10.0.0.9 -P 22 -pw "密码" -batch "ls -la" & "$Dir\..\tools\pscp.exe" -pw "密码" -P 22 ".\app.jar" user@10.0.0.9:"/opt/app/"
注意：Computer Control（改 IP / DNS）、重启网卡等操作需要以管理员身份运行本程序；plink / pscp 的密码会出现在命令行参数中（进程列表可见），仅在可信机器上使用。
